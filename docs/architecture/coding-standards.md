# Coding Standards

## Python (Backend / ML)
*   **Formatter:** Black
*   **Linter:** Flake8 / Ruff
*   **Type Hinting:** Required for all function signatures.
*   **Docstrings:** Google Style required for all public modules and functions.
*   **Testing:** Pytest.

## TypeScript (Frontend)
*   **Linter:** ESLint
*   **Formatter:** Prettier
*   **Strict Mode:** Enabled (`strict: true` in tsconfig).
*   **Component Style:** Functional Components with Hooks.
*   **State Management:** React Context or Zustand (avoid Redux unless necessary).

## General
*   **Commits:** Conventional Commits (e.g., `feat: login feature`, `fix: crash on input`).
*   **Branches:** `feature/description` or `fix/description`.
